Priority 1 — M46 T1: GLR Loop in Emitted Parser
What is missing.

The emitted parser.c parse loop takes the first action and stops. Tree-sitter’s runtime forks into multiple concurrent stack versions for states with multi-action entries, prunes by error cost and dynamic precedence, and accepts on the first surviving version that reaches the augmented end state.

The TSUnresolved side tables are already emitted (M45 Phase 2). The behavioral harness already encodes the full algorithm. What remains is translating that algorithm into the C template emitted by parser_c.zig.

Implementation shape (from docs/milestones/MILESTONES_46.md T1).

Context struct — add to the emitted parser context:

#define TS_MAX_VERSIONS 6
typedef struct { uint16_t *stack; uint16_t stack_len; uint32_t cursor;
                 int32_t  dynamic_precedence; uint32_t error_count; } TSVersion;
TSVersion ts_versions[TS_MAX_VERSIONS];
uint16_t  ts_version_count;

Copy
Outer-inner loop shape — replace the current linear advance loop with:

do {
  for each active version: {
    lex one token at version.cursor;
    look up (state, token) in parse table;
    if no action: attempt error recovery on this version;
    if shift: push new state, advance cursor;
    if reduce: pop N, lookup goto, push new state;
    if accept: record accepted version;
    if TSUnresolved entry: fork one new version per extra action;
  }
  condense_versions();  // merge identical (state, cursor) pairs
} while (version_count > 0 && !accepted);
c
Copy
Fork — when ts_parser_unresolved(state_id) returns a non-null entry: for each extra action beyond the primary, copy the current version into a new slot (up to TS_MAX_VERSIONS) and apply the alternative action there.

condense_versions() — after each full lookahead step, scan all pairs; for any two versions with identical cursor and identical stack top, keep the one with higher dynamic_precedence and discard the other.

Dynamic precedence — accumulate dynamic_precedence from each reduce production (carried in the serialized production table). Use it as the tiebreaker in condense_versions.

Error recovery in the loop — port the harness recoverFromMissingAction logic: stage-1 (scan backward for a state that handles the lookahead), stage-2 (skip one byte/token). Increment error_count on any recovery. Prune versions whose error_count exceeds the minimum surviving version by more than a threshold (tree-sitter uses 3).

Wire into behavioral harness — add a flag to run both the existing harness path and the emitted-C path on the same input, asserting identical accepted/rejected outcome for all shortlist sample inputs. This catches divergence without waiting for a full compat run.

Files. src/parser_emit/parser_c.zig (loop template), src/parser_emit/parser_tables.zig (dynamic_precedence in production table), src/parse_table/serialize.zig (confirm dynamic_precedence is serialized), src/compat/harness.zig (dual-path gate test).

Gate.

All current compat shortlist targets pass through the new loop.
The behavioral harness dual-path test passes for all shortlist grammars.
Grammars without any TSUnresolved entries (the majority) show no behavioral change — they never fork, so the outer loop is a no-op.
Risk. High — touches the generated parse loop. Develop behind -Dglr-loop=true build flag so the existing single-action path keeps compiling and passing tests throughout development.

Priority 2 — M46 T2 Remainder: Error Recovery in Emitted Parser
Prerequisite. T1 (GLR loop) must be complete and passing.

What is missing.

The behavioral harness two-stage error recovery exists and is tested. The emitted parser.c has no RECOVER action and produces no ERROR nodes — invalid input causes a hard parse failure with no partial tree.

Implementation shape.

RECOVER action in table — add ts_action_recover constant to the emitted C action enum.

ERROR node construction — when all versions fail (no surviving path), emit an ERROR node spanning the skipped bytes and restart parsing from the recovery state.

Test — parse a grammar with deliberate invalid input and assert:

a partial tree is produced (not a hard failure)
the tree contains an is_error = true node spanning the bad input
valid input following the error is correctly parsed into a sibling node
Files. src/parser_emit/parser_c.zig, src/parser_emit/compat.zig, src/behavioral/harness.zig (integration assertion).

Gate. Error recovery behavioral test from M46 passes against the emitted parser, not just against the harness simulation.

Prerequisite. M47 tree builder (ParseNode + value stack) from next-steps-260426.md Priority 4 must be merged.

What is missing.

The tree builder produces a complete ParseNode tree per parse. What remains is the reuse logic: threading an old tree into a new parse and skipping re-parsing of unchanged subtrees.

simulateIncremental entry point — new function in harness.zig accepting old_tree: ?ParseNode and edit: Edit alongside the grammar and input. Delegates to the existing GLR loop with one additional check at each advance step.

Reuse check in the advance loop — before lexing at cursor:

if old_tree has a subtree node N such that:
  N.start_byte == cursor
  AND cursor >= edit.new_end_byte   (node starts at or after edit end)
  AND parse_state == N.entry_state
then reuse N: advance cursor to N.end_byte, push N's reduction
