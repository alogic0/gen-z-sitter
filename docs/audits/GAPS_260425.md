## Current Remaining Gaps

### 1. Heavy C Parse-Table Construction Cost

`tree_sitter_c_json` now completes the routine compatibility path, including coarse
serialization, parser-table emission, parser.c emission, compatibility validation, and
compile-smoke. It is still expensive enough that it should stay in the heavy
compatibility step:

- serialize path is about 2-3 minutes locally
- peak RSS is roughly 3.4 GB
- it is not suitable for the fast default `zig build test` path

The next improvement should target parse-table memory pressure, especially item-set and
lookahead storage in coarse construction mode.
